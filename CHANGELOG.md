## Release Notes

Releases are automatically generated from commit messages using [Conventional Commits](https://www.conventionalcommits.org/).

### Commit Format

Use the following format for your commits to automatically categorize changes:

```
feat(scope): description        → Added feature
fix(scope): description         → Bug fix
docs(scope): description        → Documentation
style(scope): description       → Code style changes
refactor(scope): description    → Code refactoring
perf(scope): description        → Performance improvement
chore(scope): description       → Maintenance tasks
ci(scope): description          → CI/CD changes
```

### Examples

```
feat(docker): add health checks for all services
fix(compose): correct volume mount paths
docs(readme): update deployment instructions
chore(deps): update base image versions
```

Breaking changes must be indicated with `BREAKING CHANGE:` in the commit body.
